# 程序化页面模板（功能文档 SEO/GEO）

此模板用于**功能输出文档页**，不是纯营销内容页。

## Frontmatter

```yaml
---
title: "<功能名 + 用户意图>"
description: "<适用人群 + 结果 + 平台>"
---
```

## 必备结构

1. `快速答案（AI 搜索）`
: 1-2 句可被 AI 搜索直接引用的答案。
2. `适用场景`
: 明确何时使用、何时不适用。
3. `步骤 / 命令`
: 可复制执行的示例。
4. `常见问答`
: 至少 3 个问句式问题。
5. `相关链接`
: 快速开始、更新日志、故障排查。

## GEO 检查清单

- 实体命名统一（`Codex CLI`、`Claude Code`、`Gemini CLI`、`MCP`、`ContextDB`）
- 单页聚焦一个主意图
- 行为有系统差异时同时给 macOS/Linux 与 Windows 命令
- 长解释前先给短答案
- 功能行为变化时必须带更新日志链接

## 内链规则

每个功能页必须链接到：
1. 一个安装/上手页
2. 一个故障排查页
3. 一个更新日志/发布记录
