# 小红书运营助手 - 设计文档

**日期**: 2026-03-01
**版本**: 1.0

## 1. 项目概述

**项目目标**: 将 Claude Code 打造成小红书运营助手，通过 Chrome DevTools MCP 控制浏览器执行运营任务，并将学习到的技能、规范和任务记录存储在 aios 文件夹中。

**核心价值**:
- 你直接给我下达任务，我理解后执行
- 我会记住你教的技能和规范，持续学习优化
- 每次操作都有记录，可追溯可复盘

## 4. 功能模块

### 4.1 任务理解与执行

- 理解自然语言任务意图
- 分解任务步骤
- 调用 Chrome DevTools MCP 执行

### 4.2 记忆系统

- **skills/**: 存储你教我的技能（如"如何写爆款标题"）
- **specs/**: 安全操作规范（如"每天最多发3篇，互动间隔随机5-15秒"）
- **history/**: 每次操作的详细记录
- **knowledge/**: 知识库（热门话题、敏感词等）

### 4.3 行为随机化

- 操作时间间隔随机化（模拟真人）
- 动作序列微调
- 异常检测与自动暂停

### 4.4 任务管理

- 口头任务 → 解析 → 执行 → 记录
- 支持任务状态跟踪（pending/done/failed）
- 失败重试机制

## 5. 运营能力

| 能力 | 说明 |
|------|------|
| 发布笔记 | 文字、图片、视频笔记 |
| 互动操作 | 点赞、评论、关注、收藏 |
| 私信回复 | 自动回复粉丝私信 |
| 数据分析 | 账号数据、笔记数据 |
| 搜索发现 | 热门内容、竞品分析 |

---

## 6. 安全规范（初始版本）

- 操作间隔: 随机 5-30 秒
- 单日上限: 发布 3 篇，互动 50 次
- 异常处理: 检测到登录失效自动暂停
- 完整日志: 所有操作记录在案
